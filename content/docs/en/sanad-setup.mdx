**Sanad** (سند) is Nassila's passage-grounding worker. It compares sentences in your draft to source excerpts using an OpenAI-compatible LLM endpoint you configure — **local by default**, cloud optional.

<Callout variant="warning" title="AI-assisted">
Sanad output is **AI-assisted**. Invalid quotes are blocked by guardrails, but you must verify every verdict manually before submission.
</Callout>

## Model downloads

Sanad GGUF files are **not bundled** in the installer. Download from Hugging Face (bring your own file):

| Tier | Model ID | VRAM (approx.) | Hugging Face |
|------|----------|----------------|--------------|
| Default | `nassila-sanad-e4b` | ~8 GB (Q6_K) | [nassila-sanad-e4b](https://huggingface.co/QinEmPeRoR93/nassila-sanad-e4b) |
| Quality | `nassila-sanad-12b` | ~12 GB+ (Q6_K) | [nassila-sanad-12b](https://huggingface.co/QinEmPeRoR93/nassila-sanad-12b) |

In Nassila: **Settings → Passage grounding** — pick **E4B** or **12B** tier chip; model field auto-fills when using a **local** Nassila GGUF.

## Configure in Nassila

<Steps>
<Step title="Open Passage grounding settings">
**Settings → Passage grounding** (or the in-app **Set up Sanad** link — full detail lives here on the website).
</Step>
<Step title="Choose local or cloud">
Under **Provider**, pick a **local** runner (LM Studio, Ollama, vLLM, Custom) for Nassila Sanad GGUFs, or **Cloud API** for an experimental third-party model.
</Step>
<Step title="Set base URL and model ID">
Base URL must match your runner or API. Model ID must match what the server exposes (`ollama list`, LM Studio loaded model, or cloud model name).
</Step>
<Step title="Test connection">
Use **Test connection** in settings before your first audit. Fix base URL, model id, or API key until it passes.
</Step>
<Step title="Enable in Manuscript">
Toggle Sanad on the **Sanad bar** when running a manuscript audit.
</Step>
</Steps>

## Runner and cloud setup

Pick the tab that matches your provider. Local tabs use Nassila-trained Sanad models; **Cloud API** uses whatever model your vendor exposes (quality not guaranteed).

<Tabs>
<Tab label="LM Studio (recommended)">

<Steps>
<Step title="Install LM Studio">
Download from [lmstudio.ai](https://lmstudio.ai/).
</Step>
<Step title="Load a Sanad GGUF">
Download **nassila-sanad-e4b** or **nassila-sanad-12b** from Hugging Face (table above) and load the file in LM Studio.
</Step>
<Step title="Start Local Server">
In LM Studio, start the **Local Server** (OpenAI-compatible API on port 1234 by default).
</Step>
<Step title="Match Nassila settings">
Preset **LM Studio**. Base URL:

<CodeBlock>http://localhost:1234</CodeBlock>

Model field: `nassila-sanad-e4b` or the exact id LM Studio shows for the loaded GGUF.
</Step>
</Steps>

</Tab>
<Tab label="Ollama">

<Steps>
<Step title="Install Ollama">
Download from [ollama.com](https://ollama.com/).
</Step>
<Step title="Pull from Hugging Face">
Default tier:

<CodeBlock>ollama pull huggingface.co/QinEmPeRoR93/nassila-sanad-e4b:Q6_K</CodeBlock>

Quality tier:

<CodeBlock>ollama pull huggingface.co/QinEmPeRoR93/nassila-sanad-12b:Q6_K</CodeBlock>

Verify the `:Q6_K` tag matches the file name on the model card.
</Step>
<Step title="Match Nassila settings">
Preset **Ollama**. Base URL:

<CodeBlock>http://localhost:11434</CodeBlock>

Model field must match **`ollama list`** exactly after pull.
</Step>
<Step title="Already have a .gguf on disk?">
Create a Modelfile with `FROM ./your-file.gguf`, then run `ollama create` with a name you will use in Nassila's model field.
</Step>
</Steps>

</Tab>
<Tab label="vLLM">

<Steps>
<Step title="Serve a Sanad GGUF">
Follow [vLLM docs](https://docs.vllm.ai/en/latest/) (or another OpenAI-compatible server) to serve **nassila-sanad-e4b** or **nassila-sanad-12b** for your GPU/RAM.
</Step>
<Step title="Note the model id">
Your server exposes a model id string in its `/v1/models` response — copy that exactly.
</Step>
<Step title="Match Nassila settings">
Preset **vLLM** or **Custom**. Base URL is your server's root (often `http://localhost:8000`). Model field = id from the server.
</Step>
</Steps>

</Tab>
<Tab label="Custom endpoint">

For any **local** OpenAI-compatible server (llama.cpp server, text-generation-webui, etc.):

<Steps>
<Step title="Start your server">
Ensure `/v1/chat/completions` is available on your base URL.
</Step>
<Step title="Load a Nassila Sanad GGUF">
Use one of the Hugging Face downloads above — Custom is not a substitute for Sanad weights.
</Step>
<Step title="Match Nassila settings">
Preset **Custom**. Enter your base URL and the model id your server reports. For **localhost**, any short non-empty API key placeholder is accepted.
</Step>
</Steps>

</Tab>
<Tab label="Cloud API">

<Callout variant="warning" title="Experimental — not Nassila Sanad">
Cloud models are **not** Nassila-trained Sanad checkpoints. Grounding quality is **experimental**. Prefer local E4B/12B GGUFs for submission-critical work.
</Callout>

<Steps>
<Step title="Choose a cloud provider">
Nassila accepts any OpenAI-compatible HTTPS endpoint. Common choices: [OpenAI](https://platform.openai.com/) or [OpenRouter](https://openrouter.ai/) — the app may auto-detect these from the base URL.
</Step>
<Step title="Preset Cloud API in settings">
**Settings → Passage grounding → Provider → Cloud API**.
</Step>
<Step title="Paste API key">
Enter your provider API key. It is stored **only on this computer** and sent **only to the endpoint you configure** — not to Nassila servers.
</Step>
<Step title="Set base URL and model">
If not auto-filled: base URL examples:

<CodeBlock>https://api.openai.com/v1</CodeBlock>

<CodeBlock>https://openrouter.ai/api/v1</CodeBlock>

Model field = the provider's model name (e.g. a chat model your account can access). **Tier chips (E4B/12B) apply to local Sanad only** — cloud uses your chosen model as-is.
</Step>
<Step title="Review privacy before auditing">
Manuscript passage text is sent to **your cloud vendor** when grounding runs. Read their retention and training policies for sensitive drafts.
</Step>
</Steps>

</Tab>
</Tabs>

## Privacy

- **Local runners** — passage text goes to your machine only (localhost or your LAN server).
- **Cloud API** — passage text goes to the vendor you configure.
- **Nassila** never receives manuscript text for grounding; L1/L2 registry checks and OA fetch are separate actions you trigger.

## Troubleshooting

| Problem | What to try |
|---------|-------------|
| Connection refused | Confirm runner server is running; check base URL port |
| Model not found | Match model id to LM Studio loaded model or `ollama list` |
| Slow or OOM | Switch to **E4B** tier; close other GPU apps |
| Empty grounding | Enable Sanad toggle; confirm source excerpt is not `Unavailable` |
| Cloud test fails | Verify API key, `/v1` suffix on base URL, and model name with provider |
