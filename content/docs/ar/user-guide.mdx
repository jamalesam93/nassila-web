لجولة خطوة بخطوة مع تفاصيل اللوحات، راجع [دليل الإجراءات](/docs/how-to).

**ناسيلا** (Nassila) تطبيق سطح مكتب لـ**بناء قائمة مراجع** و**التحقق** منها وفق قواعد CSL، وتشغيل **فحص سجل موحّد** (L1 + L2)، و**التصدير** لقائمة مراجع منسّقة قبل التسليم.

*تحقق من مراجعك. أسّس ادعاءاتك.*

## سير العمل الرئيسي

<Steps>
<Step title="أضف المراجع">
الصق نصاً عادياً أو BibTeX أو RIS أو CSL-JSON في لوحة **Input**، أو استخدم **Import** (شريط الأدوات أو Ctrl+I) لـ `.bib` أو `.ris` أو `.json` أو `.docx` أو `.pdf` (يستخرج DOCX/PDF قوائم المراجع حيث أمكن).
</Step>
<Step title="حل المعرفات">
استخدم الصف العلوي في Input للصق DOI أو PMID أو URL أو معرفاً واحداً لكل سطر، ثم **Resolve**.
</Step>
<Step title="أصلح المشاكل المحلية">
يسرد الشريط الجانبي **Issues** مشاكل التحقق ومجموعات التكرار و(بعد التحقق) عدم تطابق حقول السجل. استخدم **Autocorrect** وإجراءات كل صف في **Output**.
</Step>
<Step title="تحقق من المراجع">
زر شريط الأدوات، أو **References → Verify references (L1/L2)…**، أو **Ctrl+Shift+V** (عند الاتصال). مرور شبكة واحد لكل صف ذي أولوية (حتى 200 عنصر): يحل **L1** عبر Crossref أو PubMed أو OpenAlex؛ يقارن **L2** بياناتك الوصفية. تظهر الفروق المتبقية كبطاقات عدم تطابق.
</Step>
<Step title="اختر النمط وصدّر">
اختر نمط CSL في الشريط الجانبي، ثم **Export** (Ctrl+E).
</Step>
</Steps>

## تثبيت المقتطفات (سند)

يتحقق سند من ادعاءات المخطوطة مقابل مقتطفات المصدر باستخدام LLM محلي ويعيد أحكام JSON منظّمة مع اقتباسات حرفية.

<Callout variant="warning" title="بمساعدة الذكاء الاصطناعي">
سند **بمساعدة الذكاء الاصطناعي**. تحقق دائماً من الأحكام يدوياً. تبقى فحوصات السجل الحتمية L1/L2 مرجعية لبيانات المراجع الوصفية.
</Callout>

### ملخص الإعداد

1. افتح **Settings → Passage grounding**.
2. اختر مشغّلاً محلياً (LM Studio أو Ollama أو vLLM أو Custom) أو **Cloud API**.
3. اختر رقاقة المستوى: **E4B** (افتراضي، ~8 GB VRAM) أو **12B** (جودة، ~12 GB+ VRAM).
4. يُملأ حقل النموذج تلقائياً بـ `nassila-sanad-e4b` / `nassila-sanad-12b`.

يظهر نافذة **Set up Sanad** عند أول استخدام مع روابط إلى Hugging Face وتنزيلات المشغّل وأوامر Ollama pull. الخطوات الكاملة: [إعداد سند](/docs/sanad-setup).

### شريط سند في المخطوطة

عند تحرير مخطوطة، يظهر **شريط سند** في الأعلى مع:

- **Toggle** — تفعيل/تعطيل التثبيت للجلسة الحالية
- **Tier chip** — التبديل بين E4B و12B
- روابط **Setup / Configure** — إعادة فتح نافذة إعداد سند أو إعدادات Passage grounding

### جلب المصادر مفتوحة الوصول (Unpaywall)

لمراجع **DOI**، يمكن لناسيلا سؤال [Unpaywall](https://unpaywall.org/products/api) عما إذا كان النص الكامل مفتوح الوصول موجوداً (بالإضافة إلى Europe PMC وملخصات السجل).

1. افتح **Settings → General → Manuscript source fetch**.
2. أدخل بريدك الإلكتروني مرة واحدة (سياسة واجهة Unpaywall API). يُحفظ **فقط على هذا الجهاز** ويُرسل **مباشرة إلى Unpaywall** من التطبيق — وليس إلى خوادم ناسيلا.
3. تدقيقات المخطوطة المستقبلية تعيد استخدامه تلقائياً.

إذا لم يُضبط البريد، تستمر الحلقة باستخدام **ملخصات السجل** و**Europe PMC** حيث أمكن؛ يُتخطى مسار Unpaywall OA فقط.

**هذا ليس تسجيل دخول جامعي.** عنوان `@institution.edu` هو جهة اتصال واجهة Unpaywall API فقط. **لا** يفتح ملفات PDF خلف جدران الدفع عبر اشتراك مكتبتك.

## المراجع قبل تدقيق المخطوطة

إذا كان قسم **References** فوضوياً أو لم يُتحقق منه أبداً، ستكون نتائج التدقيق ضعيفة (ربط استشهاد→مرجع سيئ، أعلام L1 خاطئة، مدخلات سند رديئة). **نظّم في Bibliography أولاً:**

1. انتقل إلى **Bibliography** في الشريط العلوي.
2. استورد أو الصق المراجع؛ شغّل **Verify references** وأصلح المشاكل في **Output**.
3. عد إلى **Manuscript**، تأكد أن قائمة المراجع المضمّنة تطابق مكتبتك المنظّمة، ثم **Run audit**.

راجع [جسر المراجع](/docs/bibliography-bridge) لإرسال المراجع بين الوضعين.

## حلقة المخطوطة

يتضمن عرض Manuscript **حلقة متكاملة**: الصق أو ارفع مخطوطة (مع قسم References)، شغّل **Run audit**، ثم راجع كل مصدر مستشهد به على اليمين.

### ما تراه بعد التدقيق

لكل استشهاد داخل النص، وسّع **Passage grounding** للمراجعة:

| اللوحة | المعنى |
|-------|---------|
| **Your passage** | نص من مخطوطتك حول الاستشهاد (ما يحكم عليه سند) |
| **Source excerpt** | مقتطف من العمل المستشهد به (ملخص أو نص OA) المُرسَل فعلياً إلى النموذج |
| **Claims + quotes** | حكم لكل ادعاء واقتباسات حرفية من المقتطف عند الدعم |
| **Source text** (العنوان) | `Abstract only` أو `Open-access full text` أو `Unavailable` — ما تم جلبه |

تلخّص رقائق **L1 / L2 / L3** تطابق السجل ومحاذاة البيانات الوصفية وتثبيت المقتطفات لذلك المرجع.

### تسميات التغطية

- **Abstract only** — الورقة الكاملة غير متاحة عبر OA؛ قارن سند مقابل ملخص السجل.
- **Open-access full text** — استُخدم HTML أو JATS من Europe PMC أو Unpaywall (قد يكون المقتطف مقتطعاً).
- **Source text unavailable** — لا ملخص ولا نص OA؛ فحوصات المقتطفات محدودة.

**إرفاق PDF** للمستشهد به غير مستخدم في التدقيقات بعد (مخطط لعامل **Masdar**).

- **Tasnif** (التصنيف) و**Sharh** (الشرح) يظهران مضمّنان في تفاصيل الحلقة ودرج المراجع — وليسا كعلامات تبويب منفصلة.
- **Raqim** (الترقيم) يعمل في وضع المراجع.
- يعمل التثبيت تلقائياً عبر سند عند تشغيل تدقيق الحلقة.
- مراحل خط الأنابيب غير المنفّذة بعد (Maktab وMasdar وShahid) تظهر كهياكل صادقة، وليست تطبيقات وظيفية.

## L1 وL2 (وضع المراجع)

- **L1 (حل السجل)** — هل استطاع التطبيق ربط المرجع بإدخال كتالوج موثوق (حل DOI/PMID، تطابق OpenAlex، أو نتائج «رمادي / غير كافٍ» صريحة).
- **L2 (محاذاة البيانات الوصفية)** — هل تتطابق حقولك مع السجل القانوني عندما يجد L1 واحداً (إشارات العنوان/السنة/المجلة/المجلد/الصفحة).

**L3** (تثبيت المقتطفات) يعمل في **حلقة المخطوطة** عند تفعيل Passage grounding. وضع Bibliography فقط يستخدم L1+L2. التفاصيل: [التحقق](/docs/verification).

## الشبكة والخصوصية

قد تشمل الاتصالات الصادرة **Crossref** و**PubMed/NCBI** و**OpenAlex** و**Unpaywall** (عند حفظ بريد في Settings → General) و**Europe PMC** لجلب مصادر المخطوطة. يعمل تثبيت LLM المحلي (سند) على جهازك عبر LM Studio أو Ollama أو vLLM أو Cloud API تضبطه. لا تلصق أسراراً في حقول الاستشهاد.
